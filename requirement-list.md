# SlideGuard MVP 需求清单（评审稿 v0.2）

> 本文用于明确 MVP 的开发与验收范围。除已从 P0 删除的 R018 外，R001—R039 均按 P0 实现。R039“自定义检查”已由原 PRD 的 P1 正式调整为 P0；为保持需求追溯关系，后续编号不递补。

## 统一约定

1. **标准值来源优先级**：用户本次扫描选择的规则集 > PPT 主题/母版中可识别的规范 > 内置“通用商务 PPT”默认规则。检测结果必须显示实际值、采用的标准值及标准值来源。
2. **问题输出要求**：每个问题必须包含规则编号、页面、对象（如适用）、严重级别、实际值、建议值、判断依据、是否可自动修复和处理状态。同一对象被多个规则命中时可分别记录，但界面应关联展示；同一事实不得由同类规则重复报错。
3. **页面类型**：优先使用 PowerPoint 版式标识；无法取得或同一版式差异过大时，根据标题区、页脚区和主要元素分布聚类。至少区分封面、目录、章节页、正文页和结束页。跨页比较仅在同类页面中进行。
4. **对象范围**：默认检查文本框、占位符、图片、形状、组合对象、表格、图表及母版/版式中实际显示的对象。不支持完整解析的对象应保留显示、不得修改，并按页汇总提示其类型和数量。
5. **数值单位**：页面位置和尺寸内部统一使用 PowerPoint point（pt），界面可同时显示厘米；颜色使用 sRGB；比例和面积阈值均以对象或页面的可见区域计算。
6. **默认阈值**：本文中的默认值必须保存在带版本号的本地规则集中，并允许后续版本调整。用户忽略或加入例外不会改变原始扫描事实和历史报告。
7. **自动修复原则**：只修复目标值唯一、不会改变文字语义且可生成新文件的问题。不能确定目标值、可能遮挡内容、需要删减内容或涉及审美判断时，仅给出建议。
8. **隐私边界**：运行期间不得建立任何外部网络连接；不得加载在线字体、图片、脚本或更新信息。本机进程间通信可以使用回环地址，但必须在隐私声明中列出用途。

|---|---|---|---|---|
| R001 | 文件导入与解析 | 导入PPT文件 | 支持点击“打开文件”和拖拽导入单个.pptx文件；导入后显示文件名、绝对路径、大小和页数。对.ppt、.pptm、加密、受保护、损坏或无法解析的文件显示具体原因且不得进入扫描 | 9.1 |

| R003 | 质检模式 | 标准检查 | 执行 R004—R017、R019—R021 的全部检测规则；快速检查已经执行的规则不得重复生成问题。标准检查额外启用完整布局检查、跨页面一致性、内容密度、异常元素和异常页面识别 | 9.2.2 |

| R005 | 文档基础健康检查 | 空白页面检查 | 页面除母版背景、页码、Logo和页脚外无其他可见内容时标记为“疑似空白”。封面、章节页、结束页、隐藏页不自动判为空白；自动删除前必须逐页列出并由用户确认 | 9.3.2 |
| R006 | 文档基础健康检查 | 页面外元素检查 | 识别可见内容全部位于画布外，或有效内容因越界被裁切的对象。允许背景、装饰线、出血图片越界；越界不影响可见内容时仅记为建议项。旋转对象按实际可见包围区域计算 | 9.3.3 |
| R007 | 文本规范检查 | 字体一致性检查 | 中文、英文及其他可识别文本统一以微软雅黑为标准字体；所有非微软雅黑字体均判定为问题，不设置强调文字、符号或公式字体例外。同时检查文本框内字体混用及缺失字体风险 | 9.4.1 |
| R008 | 文本规范检查 | 字号一致性检查 | 默认正文最小字号为14pt、注释和页脚为10pt；同层级字号与主流值相差超过2pt时提示。上下标、数学公式、局部强调和图表数据标签按各自类型比较，不与正文混合判断 | 9.4.2 |
| R009 | 文本规范检查 | 文本颜色检查 | 标准色来自当前规则集；未配置企业标准色时使用PPT主题色。正文与背景对比度默认不得低于4.5:1，大号文字不得低于3:1；单页正文文本颜色超过4种时提示，图片、渐变、透明装饰和超链接状态色不计入 | 9.4.3 |
| R010 | 文本规范检查 | 文本溢出检查 | 检查文本是否被截断、超出文本框、因自动缩小低于该类型最小字号或超出页面。必须按软件内置排版结果判断并记录所用字体；因字体缺失导致结果不可靠时同时标记“字体缺失风险” | 9.4.4 |
| R011 | 版面布局检查 | 元素对齐检查 | 对同组或重复布局中的元素检查左、右、顶、底、水平中心和垂直中心对齐。默认位置容差为3pt；“同组”优先采用PPT组合关系，其次采用相同类型、相似尺寸及规律排列识别 | 9.5.1 |
| R012 | 版面布局检查 | 元素间距检查 | 对至少3个规律排列的同组对象比较相邻间距；各间距与中位数偏差超过3pt且超过中位数的10%时提示。标题正文和列表项仅在同类页面或同组对象中比较；页面边距统一由R014负责 | 9.5.2 |
| R013 | 版面布局检查 | 元素重叠检查 | 当前景对象遮挡另一对象的有效文字或主要可见内容时报告。默认重叠面积超过较小对象可见面积的10%触发；背景色块、蒙版、组合对象内部、图标底板和用户例外不报告 | 9.5.3 |
| R014 | 版面布局检查 | 页面边距检查 | 默认安全边距为页面宽高的3%，关键文字、图片主体和图表不得进入安全边距。全幅背景、Logo、页码、页脚和装饰对象例外；规则集可覆盖默认值 | 9.5.4 |
| R015 | 版面布局检查 | 页面内容密度检查 | 综合字符数、对象数、内容覆盖率、最小字号、重叠率和留白率形成0—100密度风险分；0—59正常、60—79内容偏多、80—100内容过密。该规则仅输出S4建议，不自动修复；具体归一化公式随规则集版本公开 | 9.5.5 |
| R016 | 跨页面一致性检查 | 标题位置一致性 | 仅比较同类页面；同类页面至少3页才自动推断主流值。位置容差默认3pt，尺寸误差默认5%，字号容差默认1pt；不足3页时仅使用母版或规则集标准 | 9.6.1 |
| R017 | 跨页面一致性检查 | 页码一致性 | 当规则集要求页码，或至少70%的同类正文页存在可识别页码时，检查缺失、位置、连续性和格式。默认排除封面、目录、章节页、结束页及隐藏页；支持纯数字、罗马数字和“当前页/总页数”格式 | 9.6.2 |
| R019 | 图片质量检查 | 图片比例检查 | 比较图片去除裁剪后的原始宽高比和显示宽高比；相对差异超过3%提示、超过10%判为高风险。用户明确裁剪不视为拉伸；无法取得原始尺寸时标记为不可判定 | 9.7.2 |
| R020 | 智能异常检测 | 异常元素识别 | 页面内至少3个对象在类型、尺寸和排列上形成相似组时，比较尺寸、字号、相对位置、颜色和间距；偏离组内中位数且超过对应基础规则容差的对象标记为异常。若R011/R012已报告同一事实，本规则只补充“组内异常”标签，不重复计数 | 9.9.1 |
| R021 | 智能异常检测 | 异常页面识别 | 按页面类型比较元素、文本、字体、颜色、留白、标题及页脚等特征；同类页面不少于5页时才自动计算异常度。结果仅作为S4建议并展示主要差异特征，不参与自动修复 | 9.9.2 |
| R022 | 问题列表与定位 | 问题列表 | 支持按页面、类型、S1—S4级别、可修复性和状态筛选，并搜索问题描述。状态固定为待处理、已修复、已忽略、已加入例外、修复失败、需人工处理；重新扫描后按文件指纹、规则和对象标识匹配继承忽略/例外状态 | 12.1 |
| R023 | 问题列表与定位 | 页面高亮 | 点击问题后跳转对应页面；对象级问题高亮异常对象和边界，有唯一参考位置时显示参考位置；页面级问题高亮整页并展示判断依据，不强制显示“标准位置” | 12.2 |
| R024 | 自动修复能力 | 一键修复 | 修复用户勾选且具有唯一目标值的问题。字体、字号、颜色和页码以规则集或同类页面主流值为目标；图片按中心点不变恢复原始比例；越界对象仅在可无冲突移入时修复；对齐和尺寸统一须显示基准对象；空白页必须逐页确认后删除 | 13.2 |

## 严重级别与评分口径

| 级别 | 定义 | 默认示例 |
|---|---|---|
| S1 严重 | 可能造成内容错误、泄密、文件不可用或关键内容无法展示 | 文件损坏、关键文本被截断、明确的敏感残留 |
| S2 高风险 | 明显影响可读性或交付专业性 | 正文越界、图片严重拉伸、大面积遮挡 |
| S3 一般 | 违反明确规范但不影响内容理解 | 非标准字体、标题偏移、间距不一致 |
| S4 建议 | 结果依赖统计或视觉判断，仅建议人工复核 | 页面偏密、颜色偏多、异常页面 |

总体分和维度分均为0—100。初始为100分，S1/S2/S3/S4单个问题默认分别扣20/8/2/0.5分；同一规则在同一页面的扣分上限分别为30/20/10/3分，总体扣分按 PRD 五个维度权重汇总。已忽略问题仍计分，已加入例外的问题不计分；修复后以重新扫描结果计分。规则集必须保存并在报告中公开实际扣分参数。

## 报告与导出验收口径

- HTML 报告断网打开后，摘要、筛选、缩略图和问题详情均可用，不引用任何外部 URL。
- `.xlsx` 的“扫描摘要”包含文件、版本、时间、扫描完整性、评分和统计；“问题清单”至少包含问题ID、规则ID、页码、对象ID、级别、类型、描述、实际值、建议值、标准来源、状态、可自动修复和修改建议。
- 导出文件名默认使用`原文件名_SlideGuard_yyyyMMdd_HHmmss`，不得覆盖已存在文件。

## 性能验收口径

1. 基准设备：Windows 11 x64、4核处理器、16GB内存、SSD；验收时记录准确型号和系统版本。
2. 基准样例：50页、文件不超过100MB，每页平均不少于5个文本/形状对象和1张图片，至少包含表格、图表、组合对象及母版元素；不包含视频、加密或损坏对象。
3. 计时范围：用户确认开始扫描至结果首页可交互，包含解析、规则计算和页面缩略图生成；首次运行按冷启动测试，连续执行3次取中位数。
4. “界面不冻结”定义为扫描期间窗口可移动、按钮可点击，用户操作在1秒内得到视觉反馈；取消操作在3秒内停止新增任务并进入已取消状态。
5. 200MB支持能力使用一份有效且可由PowerPoint打开的199—200MB样例验证；“正常处理”指不崩溃、不损坏源文件并能输出明确的完成或局部解析失败结果。

## 效果验收口径

1. 测试集不少于10份真实或脱敏PPT，并额外制作覆盖每条P0规则的定向样例；测试集不得参与阈值调优。
2. 由2名熟悉PPT规范的评审者独立标注，意见不一致时由第3人裁定；真值记录到页面和对象级。
3. 检出率 = 正确检出真问题数 / 真问题总数；无效问题比例 = 误报数 / 系统报告问题总数；同一对象同一事实只计一次。
4. 自动修复成功指目标问题消失、未新增S1/S2问题、文字语义不变且输出文件可被PowerPoint正常打开；成功率按执行的修复问题数计算。
5. 验收目标沿用 PRD 25.4：常见格式问题检出率≥90%，图片拉伸、页面外元素和字体异常检出率≥95%，无效问题比例≤20%，自动修复成功率≥90%。
